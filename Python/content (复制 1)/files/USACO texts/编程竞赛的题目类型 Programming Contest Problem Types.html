编程竞赛的题目类型
Programming Contest Problem Types

Hal Bruch在1999年的春假进行了一次分析，并得出了一个惊人的发现：编程竞赛的题目类型只有16种！而且，前几种在IOI的比赛中占了接近80%的比例。它们是：

  • 动态规划（Dynamic Programming）
  • 贪心（Greedy）
  • 完整搜索（Complete Search）
  • 洪水填充（Flood Fill）
  • 最短路（Shortest Path）
  • 递归搜索技术（Recursive Search Techniques）
  • 最小生成树（Minimum Spanning Tree）
  • 背包（Knapsack）
  • 计算几何（Computational Geometry）
  • 网络流（Network Flow）
  • 欧拉路径（Eulerian Path）
  • 二维凸包（Two-Dimensional Convex Hull）
  • 高精度（BigNums）
  • 启发式搜索（Heuristic Search）
  • 近似搜索（Approximate Search）
  • 特殊题目（Ad Hoc Problems）

最具挑战性的题目是组合题目，是多个算法的结合。即使它们在概念上看起来十分“显而易见”，要解决这些问题也是十分棘手的。

如果您能掌握这些题目类型中的40%，几乎可以保证您能在IOI获得一枚银牌。掌握80%几乎就能保证您在IOI超过获得金牌的分数线。当然，“掌握”是一个难以解决的问题！我们将提供大量的题目，以便您能为进入国际殿堂磨练您的技术。
