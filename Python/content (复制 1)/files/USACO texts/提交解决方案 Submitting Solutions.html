第一个挑战

最简单的编程挑战名为“test”，需要您从文件“test.in”的一行中读入一对较小的整数，输出它们的和到文件"test.out"。

下面是一个C语言的解决方案。注意使用“exit (0);”可以正确地退出程序的运行。 

/*
ID: 在这里写上您的ID
LANG: C
TASK: test
*/
#include <stdio.h>
#include <stdlib.h>
void
main () {
    FILE *fin  = fopen ("test.in", "r");
    FILE *fout = fopen ("test.out", "w");
    int a, b;
    fscanf (fin, "%d %d", &a, &b);      /* 输入的两个整数 */
    fprintf (fout, "%d\n", a+b);
    exit (0);
}

/* 使用斜杠+星号风格的注释，否则系统不会发现您的头部信息 */
/*
ID: 在这里写上您的ID
LANG: JAVA
TASK: test
*/
import java.io.*;
import java.util.*;

class test {
  public static void main (String [] args) throws IOException {
    // 用BufferedReader比RandomAccessFile更好；它快多了
    BufferedReader f = new BufferedReader(new FileReader("test.in"));
                                                  // 输入文件在上面
    PrintWriter out = new PrintWriter(new BufferedWriter(new FileWriter("test.out")));
    // StringTokenize vs. readLine/split——快多了
    StringTokenizer st = new StringTokenizer(f.readLine());
						  // 读入整行并转换成token
    int i1 = Integer.parseInt(st.nextToken());    // 第一个整数
    int i2 = Integer.parseInt(st.nextToken());    // 第二个整数
    out.println(i1+i2);                           // 输出结果
    out.close();                                  // 关闭输出文件
  }
}
重要提示：BufferedReader和StringTokenizer比其他读入方式远远有效得多。它们可以对您程序的效率造成巨大的影响！使用它们！
